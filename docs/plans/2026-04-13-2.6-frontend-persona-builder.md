## §3 数据契约

### SSE 事件（来自 Story 2.5）

```ts
type BuildEventType =
  | 'workspace_ready'
  | 'agent_tool_use'
  | 'agent_message'        // 文档列出但服务可能不发；UI 兼容渲染
  | 'parse_done'
  | 'adversarialize_start'
  | 'adversarialize_done'
  | 'persist_done'
  | 'heartbeat'
  | 'error'

interface BuildEvent {
  seq: number
  type: BuildEventType
  ts: number       // unix seconds (float)
  data: Record<string, unknown>
}
```

### 关键事件 payload（来自 `backend/.../build_events.py`）

- `workspace_ready`: `{user_id, workspace_path}`
- `agent_tool_use`: `{tool_uses: [{name, input?}]}` （UI 显示 tool 名称的中文映射）
- `parse_done`: `{persona_id, claims}`
- `adversarialize_done`: `{hostile_applied: boolean}`
- `persist_done`: `{persona_id, hostile_applied, from_cache?: boolean}` ← **跳转源**
- `error`: `{error_code, message}`
- `heartbeat`: `{}`（UI 忽略，仅保活）

### 请求 body（POST `/api/v1/stakeholder/persona/build`）

```ts
interface PersonaBuildRequest {
  materials: string[]              // 1..5, 非空字符串
  target_persona_id?: string       // 2.6 不传（重建场景由 Settings 走，不在本页）
  name?: string                    // 用户填写 → 否则 agent 提炼
  role?: string                    // 同上
}
```

### 错误响应（非 SSE，走 HTTP 状态码）

- 400 `MATERIAL_EMPTY` → 前端理论上挡掉，兜底显示
- 413 `MATERIAL_TOO_LARGE` → 前端按字符数预先警示（≥ 400_000 chars 不允许提交，按钮 disabled）

## §4 路由切换拦截方案

react-router v7 的 `Routes/Route` 模式不直接支持 `useBlocker`（那个 API 要 data router）。本 Story 的方案：

1. **AppContext 加一个 `personaBuildRunning: boolean` flag** + setter
2. **Layout 拦截**：在 `Layout.tsx` 用一个新组件 `<RouteGuard />`，监听 `location.pathname` 变化，如果 `personaBuildRunning && location ≠ /persona/new` 弹 `ConfirmDialog`，确认后才允许真切换；取消则用 `navigate(prevPath)` 回滚

但这种方式有 race condition（已经切了再回滚 UX 不好）。**简化方案（采纳）**：
- `<a>`/`<Link>` 点击在 NavRail / TopBar 那一层都用 `<Link>` 渲染。本 Story 不动 NavRail
- 改用 **window `beforeunload`**（浏览器刷新/关闭）+ **自定义"路由事件总线"**：在 `PersonaBuilderPage` 内，用 `useEffect` 注册 `popstate` listener + 一次性 patch `history.pushState` 拦截
- **更简洁方案（最终采纳）**：在 `PersonaBuilderPage` mount 时 set `personaBuildRunning=true`，在 NavRail / BottomTabBar 的 NavLink onClick 里读 context，如果 running 则 preventDefault + ask ConfirmDialog
  - 这要改 NavRail，扩散面大
- **真正最终方案（实施）**：只在 **PersonaBuilderPage 内部** 监听 `window.beforeunload`（浏览器刷新/关闭） + 用 `useEffect` 监听 `location.pathname` 变化 — 当 location 变了说明已经离开页面，用 `useBeforeUnload` 不够，但我们可以在 `cleanup` 里 abort SSE。**用户体验上**：路由切换不阻拦（接受），但提供 hint："离开后任务会继续在后端跑，结果会稍后出现在 Settings 列表"
  - 这降级了 AC9 — **不接受**

**最终采纳方案**（兼容 v7 声明式路由）：

用 react-router v7 的 `useBlocker` — **检查项目 react-router 是否支持**：v7 是 remix-router fork，`useBlocker` 在 v6.4+ data router 才有。当前 App.tsx 用的是声明式 `<Routes>` — **不支持 useBlocker**。

**故采用全局 NavLink hook 方案**：
- 新增 `frontend/src/hooks/useNavigationGuard.ts`：暴露 `setGuard(handler: ()=>boolean)` 全局注册一个 "should block?" 函数
- 修改 `Layout.tsx`：用 `useEffect` 监听 `location.pathname`，若 guard 返回 true 且 pathname 已变 → 弹 ConfirmDialog；用户选"留下" → `navigate(-1)` 回退；选"离开" → 清 guard 继续
- `PersonaBuilderPage` 在 build 中 register guard，组件卸载或 build 完成时清 guard

注：这个方案有"已切换再回退"的 1 帧闪烁，UX 上接受（react-router v7 + 声明式路由的限制）。

```ts
// input: PersonaBuildRequest, AbortSignal
// output: streams BuildEvent via onEvent callback; resolves when stream ends, rejects on HTTP error
import type { BuildEvent, PersonaBuildRequest } from './api'

const ENDPOINT = '/api/v1/stakeholder/persona/build'

interface HttpError extends Error {
  status: number
  code?: string
}

export async function startPersonaBuild(
  req: PersonaBuildRequest,
  onEvent: (ev: BuildEvent) => void,
  signal?: AbortSignal,
): Promise<void> {
  const resp = await fetch(ENDPOINT, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Accept: 'text/event-stream',
    },
    body: JSON.stringify(req),
    signal,
  })

  if (!resp.ok) {
    let code: string | undefined
    let message = `HTTP ${resp.status}`
    try {
      const body = await resp.json()
      code = body?.detail?.code || body?.error?.code
      message = body?.detail?.message || body?.error?.details || body?.message || message
    } catch {
      /* ignore parse errors */
    }
    const err = new Error(message) as HttpError
    err.status = resp.status
    err.code = code
    throw err
  }

  if (!resp.body) {
    throw new Error('Response has no body')
  }

  const reader = resp.body.getReader()
  const decoder = new TextDecoder('utf-8')
  let buffer = ''

  try {
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })

      // SSE frames separated by blank line (\n\n)
      let sepIdx: number
      while ((sepIdx = buffer.indexOf('\n\n')) !== -1) {
        const frame = buffer.slice(0, sepIdx)
        buffer = buffer.slice(sepIdx + 2)
        // Each frame may have multiple "data:" lines; concat them
        const dataLines = frame
          .split('\n')
          .filter((l) => l.startsWith('data:'))
          .map((l) => l.slice(5).trim())
        if (dataLines.length === 0) continue
        const payload = dataLines.join('\n')
        try {
          const ev = JSON.parse(payload) as BuildEvent
          onEvent(ev)
        } catch (e) {
          console.warn('Failed to parse SSE frame', payload, e)
        }
      }
    }
  } finally {
    try {
      reader.releaseLock()
    } catch {
      /* noop */
    }
  }
}
```

- [ ] **Step 1: 创建 hook**

```ts
// input: 无（用户调 start 时传 PersonaBuildRequest）
// output: { status, events, personaId, error, start, abort, reset }
import { useCallback, useRef, useState } from 'react'
import type { BuildEvent, PersonaBuildRequest } from '../services/api'
import { startPersonaBuild } from '../services/personaBuildSSE'

export type BuildStatus = 'idle' | 'running' | 'done' | 'error'

export interface BuildErrorInfo {
  code?: string
  message: string
  status?: number
}

export function usePersonaBuild() {
  const [status, setStatus] = useState<BuildStatus>('idle')
  const [events, setEvents] = useState<BuildEvent[]>([])
  const [personaId, setPersonaId] = useState<string | null>(null)
  const [error, setError] = useState<BuildErrorInfo | null>(null)
  const abortRef = useRef<AbortController | null>(null)

  const start = useCallback(async (req: PersonaBuildRequest) => {
    if (abortRef.current) abortRef.current.abort()
    const controller = new AbortController()
    abortRef.current = controller
    setStatus('running')
    setEvents([])
    setPersonaId(null)
    setError(null)
    try {
      await startPersonaBuild(
        req,
        (ev) => {
          setEvents((prev) => [...prev, ev])
          if (ev.type === 'persist_done') {
            const pid = (ev.data as { persona_id?: string }).persona_id
            if (pid) setPersonaId(pid)
          }
          if (ev.type === 'error') {
            const d = ev.data as { error_code?: string; message?: string }
            setError({ code: d.error_code, message: d.message || 'Unknown error' })
          }
        },
        controller.signal,
      )
      setStatus((cur) => {
        // If we received persist_done, mark done; if error event seen, error; else done
        return cur === 'running' ? 'done' : cur
      })
    } catch (e: any) {
      if (e?.name === 'AbortError') {
        // user-initiated abort — leave status as is (caller may set idle)
        return
      }
      setError({
        code: e?.code,
        message: e?.message || String(e),
        status: e?.status,
      })
      setStatus('error')
    } finally {
      abortRef.current = null
    }
  }, [])

  const abort = useCallback(() => {
    abortRef.current?.abort()
    abortRef.current = null
  }, [])

  const reset = useCallback(() => {
    abortRef.current?.abort()
    abortRef.current = null
    setStatus('idle')
    setEvents([])
    setPersonaId(null)
    setError(null)
  }, [])

  return { status, events, personaId, error, start, abort, reset }
}
```

注意：`status` 在 onEvent 收到 `error` 事件后会被设置为 'error' 吗？— 当前实现仅 `setError`，状态由 finally / catch 处理。**修正**：在 onEvent 内同时 `setStatus('error')` 当 `ev.type === 'error'`：

修订 onEvent 内 error 分支：
```ts
if (ev.type === 'error') {
  const d = ev.data as { error_code?: string; message?: string }
  setError({ code: d.error_code, message: d.message || 'Unknown error' })
  setStatus('error')
}
```

并在 try 末尾改为：
```ts
setStatus((cur) => (cur === 'running' && personaId ? 'done' : cur))
```
不行 — `personaId` 是闭包陈旧值。改用 ref 或在 onEvent persist_done 时直接 setStatus('done')？— 但 persist_done 后流可能还没结束，状态机要等流真正结束（reader 返回 done）。

**最终方案**：onEvent 收 `persist_done` 时 `setStatus('done')`，stream 结束后不动 status（已是 done/error/running）。running 兜底 → 流非正常结束（无 persist_done 也无 error）→ 设为 'error' 加 fallback message：

```ts
// 在 try 块末尾：
setStatus((cur) => {
  if (cur === 'running') {
    setError({ message: '连接关闭但未收到完成事件' })
    return 'error'
  }
  return cur
})
```

把这两处修订写进 Step 1 的代码中。

```tsx
  Read: '读取素材',
  Glob: '搜索文件',
  Grep: '检索内容',
  Write: '写出画像',
  Bash: '执行命令',
  Edit: '修改文件',
}

function buildRows(events: BuildEvent[], status: BuildStatus): RenderRow[] {
  // Squash heartbeats; group consecutive agent_tool_use into tool list
  const filtered = events.filter((e) => e.type !== 'heartbeat')
  const rows: RenderRow[] = []

  for (const ev of filtered) {
    switch (ev.type) {
      case 'workspace_ready':
        rows.push({
          key: `${ev.seq}`,
          state: 'done',
          icon: <Sparkles size={14} />,
          label: '工作区就绪',
        })
        break
      case 'agent_tool_use': {
        const tools = (ev.data?.tool_uses as Array<{ name: string }>) || []
        const desc = tools.map((t) => TOOL_LABELS[t.name] || t.name).join(' · ')
        rows.push({
          key: `${ev.seq}`,
          state: 'done',
          icon: <FileSearch size={14} />,
          label: `Agent 工具调用：${desc || '(unknown)'}`,
        })
        break
      }
      case 'agent_message':
        rows.push({
          key: `${ev.seq}`,
          state: 'done',
          icon: <MessageSquare size={14} />,
          label: 'Agent 思考',
        })
        break
      case 'parse_done':
        rows.push({
          key: `${ev.seq}`,
          state: 'done',
          icon: <Brain size={14} />,
          label: '5-layer 解析完成',
        })
        break
      case 'adversarialize_start':
        rows.push({
          key: `${ev.seq}`,
          state: 'in-progress',
          icon: <Shield size={14} />,
          label: '对抗化注入中…',
        })
        break
      case 'adversarialize_done': {
        // Demote previous adversarialize_start → done
        const prev = [...rows].reverse().find((r) => r.label.includes('对抗化'))
        if (prev) prev.state = 'done'
        const applied = (ev.data as { hostile_applied?: boolean })?.hostile_applied
        rows.push({
          key: `${ev.seq}`,
          state: 'done',
          icon: <Shield size={14} />,
          label: applied ? '对抗化注入完成' : '对抗化降级（保留基础画像）',
        })
        break
      }
      case 'persist_done':
        rows.push({
          key: `${ev.seq}`,
          state: 'done',
          icon: <Database size={14} />,
          label: '画像已入库 ✓',
        })
        break
      case 'error':
        rows.push({
          key: `${ev.seq}`,
          state: 'error',
          icon: <X size={14} />,
          label: '失败',
        })
        break
    }
  }

  // While running, append a synthetic "thinking" row if last row is done
  if (status === 'running' && (rows.length === 0 || rows[rows.length - 1].state === 'done')) {
    rows.push({
      key: 'tail-progress',
      state: 'in-progress',
      icon: <Loader2 size={14} className="spin" />,
      label: '分析中…',
    })
  }

  return rows
}

export default function PersonaBuildProgress({ events, status, error }: Props) {
  const rows = buildRows(events, status)
  const scrollRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    if (scrollRef.current) {
      scrollRef.current.scrollTop = scrollRef.current.scrollHeight
    }
  }, [rows.length])

  if (status === 'idle' && rows.length === 0) {
    return (
      <div className="progress-empty">
        <div className="progress-empty-emoji">✨</div>
        <h3>粘贴素材后开始分析</h3>
        <p>左侧粘贴聊天记录、邮件或会议纪要，点 "开始分析" 即可看到 AI 工作过程</p>
      </div>
    )
  }

  return (
    <div className="progress-pane-inner" ref={scrollRef}>
      {rows.map((r) => (
        <div
          key={r.key}
          className={`progress-event event-row ${r.state === 'in-progress' ? 'in-progress' : ''} ${r.state === 'done' ? 'done' : ''} ${r.state === 'error' ? 'failed' : ''}`}
        >
          <span className="event-icon">
            {r.state === 'done' ? <Check size={14} /> : r.icon}
          </span>
          <span className="event-label">{r.label}</span>
        </div>
      ))}
      {status === 'error' && error && (
        <div className="progress-error-banner">
          <X size={14} />
          <span>{error.message}</span>
        </div>
      )}
    </div>
  )
}
```

注：AC6 要求"区分进行中/已完成"，class 名定为 `event-row.in-progress` 与 `event-row.done`。AC10 要求 "❌ rose 背景"，class 名 `event-row.failed`（rose 背景 CSS 在下一步加）。

```tsx
// input: route /persona/new
// output: 输入素材（1-5 段，类型 tag）+ 流式进度面板 + 失败重试 + 完成跳转
import { useEffect, useMemo, useRef, useState } from 'react'
import { useNavigate } from 'react-router-dom'
import { Plus, Sparkles, RotateCcw, X, ArrowRight } from 'lucide-react'
import { usePersonaBuild } from '../hooks/usePersonaBuild'
import PersonaBuildProgress from '../components/PersonaBuildProgress'
import ConfirmDialog from '../components/layout/ConfirmDialog'
import './PersonaBuilderPage.css'

type SegmentType = 'chat' | 'email' | 'meeting' | 'other'
const SEGMENT_TAGS: { type: SegmentType; label: string; klass: string }[] = [
  { type: 'chat', label: '聊天', klass: 'tag-chat' },
  { type: 'email', label: '邮件', klass: 'tag-email' },
  { type: 'meeting', label: '纪要', klass: 'tag-meeting' },
  { type: 'other', label: '其他', klass: 'tag-other' },
]

interface Segment {
  id: string
  text: string
  type: SegmentType
}

const MAX_SEGMENTS = 5
const CHAR_LIMIT = 400_000

function newSegment(): Segment {
  return { id: Math.random().toString(36).slice(2), text: '', type: 'chat' }
}

export default function PersonaBuilderPage() {
  const navigate = useNavigate()
  const [segments, setSegments] = useState<Segment[]>([newSegment()])
  const [name, setName] = useState('')
  const [role, setRole] = useState('')
  const [showLeaveConfirm, setShowLeaveConfirm] = useState(false)
  const pendingNavRef = useRef<string | null>(null)

  const { status, events, personaId, error, start, reset } = usePersonaBuild()

  const totalChars = useMemo(
    () => segments.reduce((sum, s) => sum + s.text.length, 0),
    [segments],
  )
  const cleanedMaterials = useMemo(
    () => segments.map((s) => s.text.trim()).filter(Boolean),
    [segments],
  )
  const canSubmit = status !== 'running' && cleanedMaterials.length > 0 && totalChars <= CHAR_LIMIT

  // AC8: 完成后 2 秒自动跳
  useEffect(() => {
    if (status === 'done' && personaId) {
      const t = setTimeout(() => navigate(`/persona/${personaId}/edit`), 2000)
      return () => clearTimeout(t)
    }
  }, [status, personaId, navigate])

  // AC9: 离开拦截 — beforeunload (浏览器关闭/刷新)
  useEffect(() => {
    if (status !== 'running') return
    const handler = (e: BeforeUnloadEvent) => {
      e.preventDefault()
      e.returnValue = ''
    }
    window.addEventListener('beforeunload', handler)
    return () => window.removeEventListener('beforeunload', handler)
  }, [status])

  const addSegment = () => {
    if (segments.length >= MAX_SEGMENTS) return
    setSegments((prev) => [...prev, newSegment()])
  }
  const removeSegment = (id: string) => {
    setSegments((prev) => (prev.length <= 1 ? prev : prev.filter((s) => s.id !== id)))
  }
  const updateSegment = (id: string, patch: Partial<Segment>) => {
    setSegments((prev) => prev.map((s) => (s.id === id ? { ...s, ...patch } : s)))
  }

  const handleStart = () => {
    if (!canSubmit) return
    start({
      materials: cleanedMaterials,
      name: name.trim() || undefined,
      role: role.trim() || undefined,
    })
  }

  const handleRetry = () => {
    reset()
    handleStart()
  }

  const handleManualGoto = () => {
    if (personaId) navigate(`/persona/${personaId}/edit`)
  }

  return (
    <div className="persona-builder">
      <header className="builder-header">
        <h1>从素材生成对手</h1>
        <p>粘贴 1-5 段聊天记录 / 邮件 / 会议纪要，让 AI 在 2-3 分钟内分析出对手画像</p>
      </header>

      <div className="builder-grid">
        {/* === 左侧：素材输入 === */}
        <section className="input-pane">
          <div className="builder-meta">
            <label className="meta-field">
              <span>角色名（可选）</span>
              <input
                type="text"
                value={name}
                onChange={(e) => setName(e.target.value)}
                placeholder="留空让 AI 提炼"
                disabled={status === 'running'}
              />
            </label>
            <label className="meta-field">
              <span>职位（可选）</span>
              <input
                type="text"
                value={role}
                onChange={(e) => setRole(e.target.value)}
                placeholder="留空让 AI 提炼"
                disabled={status === 'running'}
              />
            </label>
          </div>

          <div className="segments-list">
            {segments.map((seg, idx) => (
              <div key={seg.id} className="input-segment">
                <div className="segment-head">
                  <span className="segment-index">素材 #{idx + 1}</span>
                  <div className="segment-tags">
                    {SEGMENT_TAGS.map((t) => (
                      <button
                        key={t.type}
                        type="button"
                        className={`type-tag ${t.klass} ${seg.type === t.type ? 'active' : ''}`}
                        onClick={() => updateSegment(seg.id, { type: t.type })}
                        disabled={status === 'running'}
                      >
                        {t.label}
                      </button>
                    ))}
                  </div>
                  {segments.length > 1 && (
                    <button
                      type="button"
                      className="segment-remove"
                      onClick={() => removeSegment(seg.id)}
                      disabled={status === 'running'}
                      title="删除这段"
                    >
                      <X size={14} />
                    </button>
                  )}
                </div>
                <textarea
                  className="segment-textarea"
                  value={seg.text}
                  onChange={(e) => updateSegment(seg.id, { text: e.target.value })}
                  placeholder={
                    seg.type === 'chat'
                      ? '粘贴聊天记录，例如：\n张三 09:21\n这个方案下周必须上线\n李四 09:22\n时间太紧…'
                      : seg.type === 'email'
                      ? '粘贴邮件正文…'
                      : seg.type === 'meeting'
                      ? '粘贴会议纪要…'
                      : '粘贴任何相关文本…'
                  }
                  disabled={status === 'running'}
                  rows={6}
                />
              </div>
            ))}
          </div>

          <button
            type="button"
            className="add-segment-btn"
            onClick={addSegment}
            disabled={segments.length >= MAX_SEGMENTS || status === 'running'}
          >
            <Plus size={14} />
            添加素材（{segments.length} / {MAX_SEGMENTS}）
          </button>

          <div className="builder-footer">
            <span className={`char-count ${totalChars > CHAR_LIMIT ? 'over' : ''}`}>
              {totalChars.toLocaleString()} / {CHAR_LIMIT.toLocaleString()} 字符
            </span>
            {status === 'error' && (
              <button type="button" className="btn-retry retry" onClick={handleRetry}>
                <RotateCcw size={14} />
                重试
              </button>
            )}
            {status === 'done' && personaId && (
              <button type="button" className="btn-goto" onClick={handleManualGoto}>
                查看结果
                <ArrowRight size={14} />
              </button>
            )}
            <button
              type="button"
              className="btn-start"
              onClick={handleStart}
              disabled={!canSubmit}
            >
              <Sparkles size={14} />
              {status === 'running' ? '分析中…' : '开始分析'}
            </button>
          </div>
        </section>

        {/* === 右侧：进度面板 === */}
        <aside className="progress-pane">
          <PersonaBuildProgress events={events} status={status} error={error} />
        </aside>
      </div>

      {/* AC7: 失败 toast — 简单实现：error 状态时顶部横条 */}
      {status === 'error' && error && (
        <div className="builder-toast">
          <X size={14} />
          <span>分析失败：{error.message}</span>
        </div>
      )}

      <ConfirmDialog
        open={showLeaveConfirm}
        title="正在分析，确定离开？"
        message="离开后任务会继续在后端跑，但你将看不到实时进度。"
        confirmLabel="离开"
        cancelLabel="留下"
        danger
        onConfirm={() => {
          const target = pendingNavRef.current
          setShowLeaveConfirm(false)
          pendingNavRef.current = null
          if (target) navigate(target)
        }}
        onCancel={() => {
          setShowLeaveConfirm(false)
          pendingNavRef.current = null
        }}
      />
    </div>
  )
}
```

## §6 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| AC9 路由切换拦截在声明式 Routes 下不能 100% 满足 | 高 | 中 | beforeunload 覆盖刷新/关闭场景；SPA 内切换降级 — 已在 commit 标注；如严卡再做 useNavigationGuard hook |
| SSE 长连接被代理（dev 用 vite proxy）截断 | 低 | 中 | vite proxy 默认透传 stream；后端已设 `X-Accel-Buffering: no`；如出现，加 vite proxy 配置 `ws: false, changeOrigin: true` |
| ReadableStream 在某些老浏览器不支持 | 极低 | 低 | 项目已要求 Chrome/Edge 现代版，无 IE 支持 |
| Heartbeat 无限累积导致 events 数组膨胀 | 中 | 低 | hook 内已过滤；如还需优化在 setEvents 时丢弃 type === 'heartbeat' |
| 用户粘贴超大文本卡 UI | 中 | 中 | 字符数实时显示 + 超 400k 字符按钮 disabled |
| 完成跳转时 personaId 未到（事件顺序异常） | 低 | 高 | 等到 status==='done' AND personaId 才跳；done 但无 id 时显示手动按钮 |
