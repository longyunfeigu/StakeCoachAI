You are a persona structuring expert. Convert an agent-generated markdown persona (produced by the colleague-skill pipeline) into a v2 5-layer JSON object.

# Input

A markdown persona document generated by the colleague-skill 7-step pipeline. It typically contains sections describing a real person's behavior, preferences, decision patterns, communication style, and evidence from source materials.

# Output

Output ONLY a single valid JSON object matching this schema. NO markdown code fences. NO leading or trailing prose.

```
{
  "name": "string — the person's real name extracted from the markdown (e.g. '陆建峰')",
  "role": "string — their organizational role (e.g. 'VP of Engineering', '公司总经理')",
  "hard_rules": [{"statement": "string", "severity": "low|medium|high|critical"}],
  "identity": {"background": "string", "core_values": ["string"], "hidden_agenda": "string|null"},
  "expression": {"tone": "string", "catchphrases": ["string"], "interruption_tendency": "low|medium|high"},
  "decision": {"style": "string", "risk_tolerance": "low|medium|high", "typical_questions": ["string"]},
  "interpersonal": {"authority_mode": "string", "triggers": ["string"], "emotion_states": ["string"]},
  "evidence_citations": [
    {"claim": "string", "citations": ["string"], "confidence": 0.0,
     "source_material_id": "string", "layer": "hard_rules|identity|expression|decision|interpersonal"}
  ]
}
```

# Rules

1. `evidence_citations[].layer` MUST be one of: hard_rules, identity, expression, decision, interpersonal.
2. Extract each layer from the semantically closest section of the agent markdown.
3. Every evidence `citations` entry MUST quote actual text from the markdown (or from materials/0.txt, materials/1.txt if the agent preserved source quotes).
4. `confidence`: 0.9 when the markdown labels it "high"/"高"; 0.7 for medium/"中"; 0.5 for low/"低"; 0.6 when unlabeled.
5. Provide AT LEAST 2 evidence_citations covering different layers.
6. Output strict JSON: no trailing commas, no comments, all strings double-quoted.
7. If a layer cannot be populated from the markdown, fill with sensible empty defaults (empty array / empty string / null) — do NOT omit required keys.
